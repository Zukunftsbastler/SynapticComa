# Art & UI

## 1. Visual Philosophy

All systems must be readable without text.

State changes must be:

* visible
* immediate
* understandable through motion and iconography

---

## 2. Action Points (AP) Representation

### 2.1 Core Concept

AP is a **shared resource** displayed centrally.

It represents:

* available actions
* current progress capacity

---

### 2.2 Visual Form

AP is displayed as:

* a set of **glass vials / energy cells**
* filled with animated fluid or light

---

### 2.3 State Changes

#### Spending AP

* fluid drains from vial
* subtle sound + visual drop

#### Gaining AP (Shared Unlock)

* strong visual pulse
* new fluid fills into vials
* glow spreads across UI

---

### 2.4 Removed Behavior

The following must NOT occur:

* ❌ automatic refill
* ❌ periodic regeneration
* ❌ “round reset” animation

AP only changes when:

* players act
* players unlock progression

---

## 3. Shared Unlock Feedback

When activated:

* both nodes glow simultaneously
* connection effect between players
* AP increase animation triggered

Goal:

> make cooperation visually obvious

---

## 4. DNA Matrix Feedback

* active paths glow
* inactive paths dim
* changes happen instantly on insert/rotate

---

## 5. Failure Feedback

### Death

* avatar shatters / dissolves

### Dead End

* UI dims
* AP display empties
* subtle “no progress” indicator

---

## 6. Design Intent

UI should communicate:

* progression
* consequence
* shared state

Never:

* hidden rules
* time-based systems

Everything must feel:

> reactive, not cyclical
